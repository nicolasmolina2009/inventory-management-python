# Project Plan

## Overview
This project aims to provide a simple inventory management system in Python while offering a learning platform for Systems Engineering students and open-source enthusiasts. The system allows users to manage product stock, track purchases and sales, and calculate profit margins.

## Objectives
- Implement a CLI-based inventory management tool using Python.
- Provide features for adding, updating, and searching products.
- Maintain purchase price and sale price for basic profit calculations.
- Allow simple reports such as inventory listing, low stock alerts, and profit analysis.
- Encourage collaboration and contributions from the open-source community to extend the project.

## Milestones
### Milestone 1 – Project Setup and Core Functionality
- Setup the project structure with modules for main CLI and inventory logic.
- Implement `Inventory` class with methods for loading and saving data to JSON.
- Implement `main.py` CLI interface with functions to add products, list products, search and update stock.
- Create `README.md` with project description and usage instructions.

### Milestone 2 – Documentation and Testing
- Write detailed documentation in `docs/` explaining code structure, usage, and contribution guidelines.
- Develop unit tests for critical functions like adding products, updating stock, and calculating margin.
- Improve error handling and user input validation.

### Milestone 3 – Enhanced Features
- Add features such as low-stock notifications and exporting reports.
- Implement basic profit and sales reporting.
- Refine CLI interface for improved user experience.
- Explore creating a simple web interface using Flask or another framework for future enhancements.

### Milestone 4 – Community Contributions
- Encourage contributions via pull requests.
- Use issues and discussions to manage feature requests and bug reports.
- Explore packaging the project for distribution on PyPI.

## Timeline
The project is planned to be developed over several weeks. Milestone 1 should be completed within the first week, Milestone 2 in the second week, and Milestones 3 and 4 depending on community interest and contributions.

## Contribution
Anyone is welcome to contribute. Please submit pull requests for bug fixes, new features, or documentation improvements. The project follows the MIT license, so please review the license file before contributing.
